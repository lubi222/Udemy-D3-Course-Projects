I also introduced the D3 update pattern in this section for building dynamic visualizations...

// JOIN new data with old elements.
var rects = g.selectAll("rect")
    .data(data);

// EXIT old elements not present in new data.
rects.exit().remove();

// UPDATE old elements present in new data.
rects
    .attr("y", function(d){ return y(d.revenue); })
    .attr("x", function(d){ return x(d.month) })
    .attr("height", function(d){ return height - y(d.revenue); })
    .attr("width", x.bandwidth);

// ENTER new elements present in new data.
rects.enter()
    .append("rect")
        .attr("y", function(d){ return y(d.revenue); })
        .attr("x", function(d){ return x(d.month) })
        .attr("height", function(d){ return height - y(d.revenue); })
        .attr("width", x.bandwidth)
        .attr("fill", "grey");

For your second project, you will be making a clone of a famous visualization from Gapminder, showing how life expectancy has been increasing over the years with GDP per capita. 

Take a look at this video here,  which shows the graph that we want to make. 

Just like last time, I’ve split up the components of this project into manageable chunks to give you some guidance and to make the process simpler. I am also providing you with a few extra hints to make the project less daunting:

  1. Take a look at the data that we're working with in your browser console. If there are any null values for one of the countries in one of the years, use a filter on the array to exclude that country-year data point from the dataset.
  2. Make a static scatter plot for the first year in our data.
    1. Set up some sensible dimensions for your visualization, and make it conform to the D3 margin convention.
    2. Write scales for each axis (GDP-per-capita on the x-axis, life expectancy on the y-axis)
      1. Suggested domains: x – [300, 150000] ; y – [0, 90] .
      2. The x scale should be a logarithmic scale.
    3. Append both axes with D3's axis generators.
      1. Use TickValues() to manually set our x-axis values of 400, 4,000, and 40,000.
    4. Append circles for each country in one year (e.g. the first year of our data)
  3. Write an update()  function that makes use of the JOIN/EXIT/UPDATE/ENTER pattern discussed in this section.
    1. The data join needs to contain a key function linking it to individual countries, or else it won’t work.
    2. Put a transition on the update function of 100ms (it can’t be larger than the d3.interval time).
  4. Add a loop with d3.interval() , calling the update()  function on each iteration of the loop.
    1. On each run of the update()  function, you should be passing in a different array of countries.
    2. The visualization should go from 1800 to 2015, then reset itself – how could you achieve this with something like our “flag” variable?
  5. Change the radius dynamically so that it represents the population of the given country in a given year.
    1. You’ll need to use another scale for this (the range should be [5, 25]).
    2. Make sure to associate population with the area, rather than the radius of the circle! You might need to play around with this scale to find the right transformation to use. (Hint: Area = PI * Radius^2)
  6. Change the fill of the circles to associate them to their continent.
    1. You’ll need to write an ordinal scale to do this. Use one of the D3 color schemes to make things simpler.
  7. Add axis labels to the x and y axis, and a year marker that updates with every run of the update loop.
